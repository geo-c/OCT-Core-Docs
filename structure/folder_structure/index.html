  • 1. Folder Structure
  • Edit on GitHub
Name Description
config This folder holds Javascript files which hold information to configure the website
controllers This folder holds all Javascript files which contain the programm logic
models This folder contains Files which define the structure of data the programm expects
public This folder cointains the website which displays information and has an User Interface to interact with the API Interface
routes Here are all routes defined which the REST-API listens to.
templates In this folder are templates for e.g. letters send by email

Files:

Name Description
package.json All included nodejs packages
server.js Here the Programm is started. To start the programm use the following command 
    $ nodejs server.js --api_port <<port>> --database_name <<name>> --postgres_user <<user>> --postgres_password <<password>> --email_user <<email>> --email_password <<email_password>>

2. controllers¶

Name Description
admin All logic regarding the admin
apps All logic regarding the apps
categories All logic regarding categories
connectors Here are connectors which connect to external databases
dataset All logic regarding datasets
logs All logic regarding the logs
main_database All logic regarding the main_database
query Here is the logic to query the database
spatial Work in Progress. In here lies the logic to scan datasets for spatial information and store them in a database
stats Here are statistical data generated and given back to the user
sub_database All logic regarding sub_database
submit Here is the logic to submit new datasets
tags All logic regarding tags
update Here is the logic to update existing datasets
